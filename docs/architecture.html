Data Flow — Filter Change
1
User selects a filter
FilterBar.vue updates the shared useFilters composable ref — e.g. selectedPeriod = "2025-02"
2
Vue watcher triggers
Each view watches [selectedPeriod, selectedLocation, selectedCategory, selectedStatus] and calls its loadData() function on change.
3
api.js builds the request
getCurrentFilters() maps composable refs to query params and fetches: GET /api/orders?month=2025-02&warehouse=all
4
FastAPI filters in-memory data
Endpoint calls apply_filters() then filter_by_month() against the in-memory list. "all" values are skipped. Returns List[Order] validated by Pydantic.
5
Vue computed properties re-derive
Raw allOrders ref is updated. Computed properties — statusData, topProducts, orderHealthMetrics — recalculate automatically and the template re-renders.
